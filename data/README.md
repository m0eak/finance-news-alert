# data/

这个目录放 `finance-news-alert` 运行过程中的中间数据和最终产物。

## 目录用途

保留每一步的输出，方便：

- 调试流程
- 对比不同版本结果
- 判断问题出在哪一层
- 回看某次运行的中间状态

## 子目录说明

- `raw/`
  - 原始抓取结果
- `normalized/`
  - 标准化后的新闻对象
- `clustered/`
  - 聚类 / 去重 / 排序后的事件簇
- `analysis/`
  - 模型分析后的结构化结果
- `output/`
  - 最终给人看的简报和提醒文本

## 推荐排查顺序

如果结果不对，建议按这个顺序看：

1. `raw/`：上游有没有抓到内容
2. `normalized/`：字段是否统一正确
3. `clustered/`：是否过度合并或排序错误
4. `analysis/`：模型输出是否跑偏或解析失败
5. `output/`：最终文本是否渲染异常

## 维护约定

- 中间文件优先保留 JSON，方便脚本继续处理
- 最终输出可用 `.txt` 或 `.md`
- 如果数据太多，可定期清理旧文件，但建议保留少量样本用于回归对比
