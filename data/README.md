# data/

这个目录放 `finance-news-alert` 运行过程中的中间数据和最终产物。

## 目录用途

保留每一步的输出，方便：

- 调试流程
- 对比不同版本结果
- 判断问题出在哪一层
- 回看某次运行的中间状态

## 子目录说明

- `raw/`
  - 原始抓取结果
- `normalized/`
  - 标准化后的新闻对象
- `clustered/`
  - 聚类 / 去重 / 排序后的事件簇
- `analysis/`
  - 提供给当前聊天模型读取的候选分析输入
- `output/`
  - 可选的最终文本输出（如 digest / alert）

## 推荐排查顺序

如果结果不对，建议按这个顺序看：

1. `raw/`：上游有没有抓到内容
2. `normalized/`：字段是否统一正确
3. `clustered/`：是否过度合并或排序错误
4. `analysis/`：候选事件是否合理、是否足够紧凑
5. `output/`：最终文本是否表达清楚

## 当前模型版说明

当前主流程不再在脚本里直接调用外部模型 API。

推荐流程是：

1. 跑脚本，生成 `data/analysis/analysis_input.json`
2. 由当前聊天模型读取该文件
3. 当前聊天模型输出：
   - digest
   - 单条 alert
   - 利好 / 利空 / 逻辑 / 置信度判断

## 维护约定

- 中间文件优先保留 JSON，方便脚本继续处理
- 最终输出可用 `.txt` 或 `.md`
- 如果数据太多，可定期清理旧文件，但建议保留少量样本用于回归对比
