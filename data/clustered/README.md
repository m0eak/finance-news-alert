# data/clustered/

这个目录存放聚类、去重、排序后的事件簇结果。

## 目录用途

这里的结果是给模型分析前最重要的一层候选事件集合。

## 这里的文件通常是什么

例如：

- 事件簇列表 JSON
- 去重后的候选新闻集合
- 带优先级分数的排序结果

## 这一层主要解决什么问题

- 同一事件多源重复
- 低质量资讯挤占上下文
- 候选新闻过多导致模型分析成本过高

## 常见内容

每个事件簇通常包含：

- 主标题
- 主来源
- 其他来源列表
- 聚合后的摘要
- 首次 / 最近出现时间
- `priority_score`
- `cluster_id`

## 使用建议

如果最终 alert 不合理，优先先看这里，判断是不是排序或聚类阶段已经跑偏。
