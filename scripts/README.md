# scripts/

这个目录放 `finance-news-alert` 的实际执行脚本。

## 目录用途

这里的文件负责把整条链路跑通：

- 抓取新闻
- 标准化
- 聚类 / 排序
- 模型分析
- 渲染输出
- 总控执行

## 当前文件说明

- `fetch_news.py`
  - 抓取国际 RSS 和国内财联社电报
  - 输出原始新闻数据到 `data/raw/`

- `normalize_news.py`
  - 把不同来源的新闻转成统一 JSON 结构
  - 做基础清洗、字段统一、标题标准化等
  - 输出到 `data/normalized/`

- `cluster_news.py`
  - 对标准化新闻做轻量聚类
  - 执行去重、合并、排序、打分
  - 输出到 `data/clustered/`

- `analyze_news.py`
  - 调用模型做结构化金融影响分析
  - 产出摘要、利好/利空方向、逻辑、影响级别、置信度等
  - 输出到 `data/analysis/`

- `render_digest.py`
  - 把分析结果渲染成最终文本
  - 生成简报和单条提醒
  - 输出到 `data/output/`

- `run_poc.py`
  - 总入口脚本
  - 串联整条 PoC 流程
  - 一般优先从这里运行

## 使用建议

- 日常手动调试：优先跑 `run_poc.py`
- 排查具体阶段问题：单独跑对应脚本
- 改字段结构时，要同步检查下游脚本是否也要改

## 维护约定

- 一个脚本尽量只负责一层职责
- 不要把抓取、分析、渲染全部混在一个脚本里
- 如果后续增加新源，优先在抓取层扩展，而不是直接改渲染层
